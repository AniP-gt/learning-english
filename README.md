# Learning English

日本語話者向けの英語学習アプリケーション（TUI + Web）。
このリポジトリは、Gemini API を活用した学習ワークフローを提供します。ローカルの TUI (Bubbletea) とブラウザベースの Web UI (Next.js) の両方から同一のコアロジックを利用できます。

このドキュメントはプロジェクトの導入手順、機能の詳細、学習方法、データ構造の推奨、および開発者向け情報をまとめた README です。

## 主要な特徴（詳細）

- 教材生成（Gemini API）: 日本語のアイデアを基に、レッスン・トピック・例文・練習問題を自動生成します。出力は Markdown とメタデータ（YAML frontmatter）で保存され、TUI / Web の双方で再利用可能です。
- 音声機能
  - STT（Speech-to-Text）: 録音した音声を Gemini/STT に送信して文字起こしを行い、正誤判定やフィードバックに利用します。
  - TTS（Text-to-Speech）: Web では Web Speech API、ローカルでは say 等を使って自然な発音を再生します。
- 画像生成: 3-2-1 画像想起や教材内イラストを画像生成 API で作成し、学習記録に埋め込みます。
- 学習フローの共有（Web）: 各セッションの記録（テキスト、音声、画像、評価）をブラウザ上で閲覧・再生・共有できます。
- 共通コアロジック（Go）: core ロジックは Go で実装され、TUI と Web 間でアルゴリズムやデータモデルを共有します。これによりクライアント実装を薄く保てます。
- 単語帳・復習機能: 単語抽出、頻度ベースの優先度、簡易 SRS（復習間隔）の導入により継続的な学習を支援します。
- 分析・メトリクス: WPM（words per minute）、発音スコア（STT 比較）、学習時間、継続日数などを収集して進捗を可視化します。
- エクスポート/インポート: Markdown とメタデータ形式でエクスポート可能。メディアファイルは Git LFS や外部ストレージに保存することを想定しています。

## リポジトリ構成（抜粋）


## クイックスタート

1. リポジトリをクローン

```bash
git clone https://github.com/<your-org>/learning-english.git
cd learning-english
```

2. 環境変数を用意（Gemini API キーなど）

```bash
cp .env.example .env
# .env を編集して GEMINI_API_KEY 等を設定してください
```

3. TUI を実行

```bash
go run cmd/tui/main.go
```

4. Web を実行

```bash
cd web
npm install
npm run dev
# ブラウザで http://localhost:3000 を開く
```

5. Docker Compose（代替）

```bash
docker compose up --build
# TUI: `docker compose run tui`
# Web: http://localhost:3000
```

## 学習ワークフロー（7 ステップ）

各ステップの目的と利用のヒントを示します。

1. Idea — 日本語でネタ出し（Gemini との対話）
   - 目的: 学習トピックの決定（例: "旅行で使うフレーズ", "ビジネスメール"）
   - ヒント: 具体的な状況・レベル・語彙ターゲットを入力すると良い結果が得られます。

2. Words — 単語帳の自動生成
   - 目的: トピックに関連する重要語と例文を抽出
   - フォーマット: words.md に単語/品詞/日本語訳/例文を保存します。

3. Reading — リーディング（WPM 計測）
   - 目的: 読解と速読練習。WPM を計測して改善を追跡します。
   - ヒント: 1 セクションを 100-200 語程度に分けると良いです。

4. Listening — 音声再生（TTS）
   - 目的: リスニング強化とシャドーイング用素材の生成
   - ヒント: 再生時に再生スピードを変え、聞き取りやすさを調整してください。

5. Speech — 録音と文字起こし（STT）
   - 目的: 発話を記録し、STT 結果と正解テキストを比較して発音フィードバックを生成します。
   - ヒント: マイク品質・ノイズ環境を整えると判定精度が上がります。

6. 3-2-1 — 画像想起トレーニング
   - 目的: 3つの項目（見たこと、気づいたこと、疑問）を画像連想で強化するトレーニング
   - ヒント: 画像生成 API を使い、視覚的な記憶を紐づけます。

7. Roleplay — 会話ロールプレイ（Gemini Chat）
   - 目的: 実践的な会話練習。ロール（店員/客、面接官/応募者 等）を指定してシミュレーションします。
   - ヒント: セッションの最後にフィードバック（改善点）を自動生成します。

## 学習方法（運用ガイド）

- 推奨サイクル: 1 トピックにつき「Idea → Words → Reading/Listening → Speech → Review」を 1 セッション（約20–40 分）で回し、翌日に復習セッションを入れることで定着を促します。
- 復習設計: 単語は簡易 SRS を用いて復習間隔を延ばす運用を推奨します。頻度は学習者の正答率で自動調整できます。
- セッション記録: 各セッションを artifacts に保存し、後で Web から振り返り・分析できるようにします。
- 評価基準: WPM（目標設定）、発音スコア（STT の文字列一致割合）、自己フィードバック（5 段階評価）を組み合わせて総合評価を算出できます。
- チーム運用: 教材・トピックは共有リポジトリで管理し、レビュー・改善を共同で行うことを推奨します。

## データ構造と運用方針（推奨）

学習データ（教材、セッション記録、メディア）はローカルの artifacts 配下に生成されますが、長期保存や共同運用を考えると「専用の別リポジトリ」を作成して管理することを強く推奨します。

理由:
- 学習データはバイナリ（音声・画像）や大きな履歴を含むため、アプリ本体のリポジトリから分離しておくと運用が楽になります。
- 複数のクライアント（TUI / Web / 他ツール）で同じデータを参照・変更する際のコンフリクト管理やアクセス制御が容易になります。

推奨レイアウト（別リポジトリ例: github.com/<your-org>/learning-english-data）:

```
learning-english-data/
├── sessions/                # セッション単位の記録
│   └── YYYY-MM-DD-sessionID/
│       ├── session.json     # メタデータ（timestamps, duration, metrics）
│       ├── topic.md         # トピック本文（YAML frontmatter + markdown）
│       ├── words.md         # 単語帳（マシン可読フォーマット）
│       ├── reading.md       # リーディング原文とメタ情報（WPM）
│       ├── transcript.txt    # STT 結果
│       └── media/           # audio/*.wav, images/*.png
├── topics/                  # 共有教材コレクション
│   └── topic-slug/
│       ├── index.md
│       └── assets/
└── README.md
```

保存の注意点:
- 大きなバイナリは Git LFS を利用するか、S3 等のオブジェクトストレージに保存して、メタデータのみを Git で管理することを推奨します。
- セッションごとにユニークな ID を付与し、session.json にチェックサムや生成元の Gemini prompt を残すと再現性が高まります。

保存先のカスタマイズ:
- デフォルトでは artifacts/ 以下に保存されますが、設定で保存先のファイルパスを指定できます。指定されたパスに対しては、week 単位で生成される Markdown ファイル（topic.md、words.md、reading.md、feedback.md）が直接書き込まれます。
- 外部リポジトリや任意のファイルパスを指定して保存する場合、ホスト側でその場所へ書き込み可能であることを確認してください。

Docker の制約:
- artifacts を別リポジトリやホスト上の任意パスに保存する設定を利用する場合、Docker Compose の設定でそのパスをボリュームマウントしていない限り、コンテナから直接その外部パスに書き込むことはサポートされません。簡潔に言うと「別リポジトリ（別ファイルパス）に保存する運用」はデフォルトの Docker 実行ではサポートされていません。
- Docker を使用する場合は、artifacts フォルダをコンテナにマウントするか、S3 等の外部ストレージを利用するワークフローを検討してください。

クライアント（TUI / Web）の利用可否:
- 本アプリの主要機能（教材生成、単語帳、リーディング、録音/STT、画像生成、ロールプレイ等）は TUI または Web のいずれからでも操作可能です。UI によって一部操作フローやメディアの表示方法が異なる場合がありますが、コアロジックは共通のため同等のワークフローを実現できます。
- Web を利用する場合、開発サーバーをローカルの IP（例: 0.0.0.0 でバインド）で起動すれば、同一ネットワーク内のスマートフォンやタブレットからもアクセスして利用できます（例: http://192.168.0.10:3000）。ローカルネットワーク外からアクセスする場合は適切なポートフォワーディングやリバースプロキシ、認証の設定を行ってください。
